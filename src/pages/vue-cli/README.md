# vue-cli

## vue文件编译过程
- 首先`webpack`中给`.vue`格式的文件增加`vue-loader`处理模块；

- 其次`webpack`增加`vueLoaderPlugin`插件；

- `vueLoaderPlugin`的主要作用是将`.vue`文件中的各语言块应用在相应的`rules`上，并增加`vueLoaderPitcher`这个`loader`给`.vue?vue`格式的文件进行处理；

- `vueLoaderPitcher`主要作用是拦截`vue`各模块，并给他们增加对应的`loader`处理模块；

```js
// webpack.config.js
const path = require('path')
const VueLoaderPlugin = require('vue-loader/lib/plugin')

module.exports = {
    mode: 'development',
    module: {
        rules: [
            // ... 其它规则
            {
                test: /\.vue$/,
                loader: 'vue-loader'
            },
            // 它会应用到普通的 `.js` 文件，以及 `.vue` 文件中的 `<script>` 块
            {
                test: /\.js$/,
                loader: 'babel-loader'
            },
            // 它会应用到普通的 `.css` 文件，以及 `.vue` 文件中的 `<style>` 块
            {
                test: /\.css$/,
                use: [
                    'vue-style-loader',
                    'css-loader'
                ]
            },
            // 它会应用到普通的 `.sass` 文件，以及 `.vue` 文件中的 `<style lang="sass">` 块
            {
                test: /\.sass$/,
                use: [
                    'vue-style-loader',
                    'css-loader',
                    {
                        loader: 'sass-loader',
                        options: {
                            indentedSyntax: true  //sass-loader 默认解析 SCSS 语言
                        }
                    }
                ]
            }
        ]
    },
    plugins: [
        // 这个插件使 .vue 中的各类语言块匹配相应的规则
        new VueLoaderPlugin()
    ]
}
```

## vue-cli编译各阶段源码分析

首先通过vue-cli创建一个项目，

1. App.vue原始文件

```html
<template>
    <div>hello</div>
</template>

<style scoped>
    div {
        color: red;
    }
<style>

<script>
import Vue from 'vue';
export default {
    data() {
        return {}
    }
}
</script>
```

2. 经过vue-loader处理之后

```js
import { render, staticRenderFns } from "./App.vue?vue&type=template&id=65a08f72&"
import script from "./App.vue?vue&type=script&lang=js&"

  script,
  render,
  staticRenderFns,
  false,
  null,
  null,
  null

)

export default component.exports
```

3. 依次对其中的依赖部分进行编译，此时各模块规则符合`vueLoaderPitcher`校验规则，进入`vueLoaderPitcher`编译部分，删除`pitcher`自身，再给各模块增加额外的`loader`处理；
